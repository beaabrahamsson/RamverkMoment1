Publicering

I denna uppgift har en publiceringsmetod som inte är FTP-baserad använts för att publicera webbplatsen. Jag har valt att använda plattformen Netlify.

Netlify

Netlify är ett webbhotell och en plattform som tillåter utvecklare att bygga högpresterande och dynamiska webbplatser, e-handelsbutiker och webbapplikationer. Netlify gör det enkalt att publicera, distributera och vara värd för en webbplats.

Då Netlify fungerar genom att använda GitHub är det enkelt att uppdatera en webbplats endast genom att göra regelbundna commits. Netlify stöder flera olika programmeringsspråk såsom HTML, CSS, JavaScript och Python men har inte stöd för exempelvis PHP eller Java.

Netlify erbjuder även en gratisversion för personliga och hobbyprojekt vilket gör det lätt att utvärdera innan man behöver betala.

Fördelar

  • Gratis att använda
  • Enekelt att komma igång och publicera en webbplats
  • Integration med Github
  • Snabbt och enkelt att uppdatera en webbplats

Nackdelar

  • Främst gjort för statiska webbplatser
  • Du behöver kunna koda för att bygga en webbplats med Netlify

Arbetssätt

För detta projekt skapade jag ett GitHub repo för alla mina filer. Efter att ha skapat ett konto hos Netlify och kopplat det till GitHub valde jag mitt repo från listan. Efter detta konfigurerade jag inställningarna för för min webbplats och valde ett passande namn. Sidan är nu live och varje gång jag gör ett commit så uppdateras webbplatsen automatiskt.
